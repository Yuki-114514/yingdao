# 影导 Android 内测清单

更新时间：2026-04-21

适用版本：
- App 名称：影导
- 包名：`com.yuki.yingdao`
- 当前测试包：`dist/yingdao-v0.1.0-internal-debug.apk`
- 版本号：`0.1.0`

测试目标：
- 验证应用能否在真实 Android 手机上正常安装、启动、授权和录制视频
- 验证“新建项目 -> 导演方案 -> 拍摄 -> 拍后反馈”主流程是否可用
- 收集不同品牌、系统版本、权限状态下的问题

已知当前版本边界：
- 已接入真实 `CameraX` 预览和录制
- 录制后会真实生成视频文件
- 拍后反馈目前仍是测试版逻辑，不是基于真实视频内容做智能分析
- 当前更适合验证流程、稳定性和权限问题，不适合评估最终 AI 质量

## 一、测试前准备

- [ ] 手机系统版本为 Android 10 或更高
- [ ] 手机有至少 500MB 可用存储空间
- [ ] 已允许安装测试 APK
- [ ] 测试前确认手机相机可正常工作
- [ ] 记录测试设备信息：
  - 手机品牌/型号：
  - Android 版本：
  - 是否首次安装：
  - 是否曾安装旧版本：

## 二、安装测试

- [ ] 成功安装 APK
- [ ] 应用图标和名称显示正常
- [ ] 首次打开应用无闪退

关注问题：
- 安装失败
- 覆盖安装失败
- 打开即闪退
- 图标或名称异常

## 三、首次启动与权限测试

- [ ] 首次启动进入首页
- [ ] 首页文案和按钮显示正常
- [ ] 点击新建项目后进入项目创建页
- [ ] 相机权限弹窗可正常出现
- [ ] 麦克风权限弹窗可正常出现

分别测试以下情况：
- [ ] 同意相机和麦克风权限
- [ ] 只同意相机，不同意麦克风
- [ ] 拒绝相机权限后，页面提示是否合理
- [ ] 去系统设置重新开启权限后，应用是否恢复正常

关注问题：
- 弹窗不出现
- 授权后仍无法预览
- 拒绝权限后页面卡死

## 四、主流程测试

### 1. 新建项目

- [ ] 点击“新建校园生活项目”
- [ ] 项目创建页默认内容显示正常
- [ ] 点击“生成导演方案”成功进入下一页

### 2. 导演方案

- [ ] 导演方案页成功生成
- [ ] 故事主线、结构节拍、镜头任务卡显示正常
- [ ] 可滚动浏览全部镜头任务卡
- [ ] 点击“开始拍摄”成功进入拍摄页

### 3. 拍摄执行

- [ ] 拍摄页能看到相机预览
- [ ] 当前镜头信息显示正常
- [ ] 点击“开始录制本镜头”后进入录制状态
- [ ] 点击“停止录制”后恢复到非录制状态
- [ ] 页面出现“拍后反馈”
- [ ] 点击“通过本镜头”后镜头状态变化正常
- [ ] 点击“重拍一条”后可重新录制
- [ ] 点击“跳过”后流程正常

### 4. 素材整理与出片建议

- [ ] 从拍摄页进入素材整理页
- [ ] 已拍素材数量和状态显示正常
- [ ] 点击生成出片建议后能进入结果页
- [ ] 标题、字幕、建议顺序正常显示

## 五、录制专项测试

每位测试同学至少完成以下 4 组录制：

- [ ] 横向缓慢移动镜头
- [ ] 静止镜头
- [ ] 近景人物镜头
- [ ] 光线较暗环境下录制

每组录制后观察：
- [ ] 录制按钮响应是否正常
- [ ] 是否能顺利停止录制
- [ ] 是否出现拍后反馈
- [ ] 应用是否卡顿
- [ ] 录制完成后，系统相册/媒体库中是否能找到视频

建议额外测试：
- [ ] 连续录制 3 条镜头
- [ ] 录制过程中切到后台再回来
- [ ] 锁屏后再解锁回到应用
- [ ] 录制很短的视频，少于 2 秒

## 六、异常与边界测试

- [ ] 启动应用后直接拒绝所有权限
- [ ] 权限关闭状态下进入拍摄页
- [ ] 录制时来电或收到系统弹窗
- [ ] 录制时切换到别的应用
- [ ] 存储空间不足时尝试录制
- [ ] 长时间停留在拍摄页后再录制
- [ ] 完成一个项目后返回首页重新创建新项目
- [ ] 应用退出重进后，最近项目是否还在

关注问题：
- 闪退
- 黑屏
- 预览卡住
- 按钮没反应
- 状态和页面不一致
- 视频录制完成但没有反馈

## 七、视觉与可用性检查

- [ ] 首页布局正常
- [ ] 中文文案没有截断或乱码
- [ ] 按钮点击区域足够大
- [ ] 页面滚动流畅
- [ ] 不同尺寸手机上没有明显布局错位
- [ ] 状态文案能看懂

重点关注：
- 小屏手机是否会遮挡按钮
- 系统字体放大后是否错位
- 深色模式下是否有不可读文字

## 八、Bug 反馈格式

建议每个问题按下面格式发到群里或表格里：

```text
问题标题：
严重程度：高 / 中 / 低
测试设备：
Android 版本：
出现页面：
复现步骤：
预期结果：
实际结果：
是否稳定复现：是 / 否
截图或录屏：
补充说明：
```

严重程度参考：
- 高：闪退、无法安装、无法录制、主流程中断
- 中：页面错乱、状态错误、反馈不正确、需要绕路才能继续
- 低：文案、样式、轻微交互问题

## 九、建议优先反馈的 5 类问题

- [ ] 安装失败或启动闪退
- [ ] 权限弹窗和权限恢复异常
- [ ] 拍摄页没有预览或无法录制
- [ ] 录制完成后没有拍后反馈
- [ ] 页面状态和实际录制结果不一致

## 十、测试结论汇总

每位测试同学最后补一句结论：

- [ ] 可正常安装
- [ ] 可正常启动
- [ ] 可正常预览
- [ ] 可正常录制
- [ ] 主流程基本可走通
- [ ] 有问题，已提交 bug

最终总体评价：
- [ ] 可以继续扩大内测
- [ ] 需要先修复关键问题再继续测试
